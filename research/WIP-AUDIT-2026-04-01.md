- **Backend-aware session resume** shipped in `369016c`: completed tasks can now resume correctly across Claude, Codex, and Gemini instead of assuming Claude-only flows.
- **OpenClaw background tasks in Agent Status** shipped in `7c0970b`: the sidebar now renders OpenClaw cron/CLI/subagent work alongside launcher and discovered agents, with cancel/notify support and graceful degradation when OpenClaw is missing.

## 2. What looks in-progress / partial / awkward

### A. Launch surfaces are materially out of sync with the actual repo

- `package.json` is now at `0.5.1-47`, but the public site, release notes, and changelog still present `0.5.1-45` as the current story.
- `site/index.html` still labels `v0.5.1-45` as the latest release and links only to `site/releases/v0.5.1-45.html`.
- `CHANGELOG.md` only has a `0.5.1-45` entry, so today’s `-46` and `-47` changes are not documented in the main release log.
- `README.md` is badly stale for an external reader: it still says auto-discovery is “Coming soon (v0.5.0)”, says `v0.4.0` onboarding is in progress, and carries a `764 passed` test badge.

### B. The roadmap is behind the code, which makes prioritization noisy

- `ROADMAP.md` still marks the whole M3.5 sort/group redesign as TODO.
- In actual code, `sortMode` exists, recency is default, grouped roots sort by freshest activity, and the summary node already shows a sort indicator.
- That means the roadmap is no longer a reliable representation of what remains; it mixes done work with genuinely open work.

### C. OpenClaw integration is real, but still clearly phase-1

- `src/services/openclaw-task-service.ts` only supports `list`, `cancel`, and `notify`.
- The tree can show blocked tasks with a “Needs Approval” label, but there is no richer blocked-state workflow yet: no `tasks show` detail fetch, no approval action, no provenance/channel context, no ACP-specific UX.
- The integration roadmap in `research/ROADMAP-OPENCLAW-INTEGRATION.md` still mostly describes future phases rather than a completed product slice.

### D. Release/readiness proof is stale

- `research/prerelease-gate/latest.json` was generated on **2026-03-30** against an older SHA, before today’s shipping commits.
- So there is no current prerelease proof artifact for the Apr 1 repo state.
