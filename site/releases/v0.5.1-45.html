Command Central Release Notes

This release wraps up today’s rollout from v0.5.1-43 through v0.5.1-45. The headline: you can now track more background work in the Agent Status sidebar, resume sessions across supported backends more reliably, and review diffs without depending on a terminal launcher.

What's New

Background OpenClaw tasks now appear in the sidebar

Command Central now brings OpenClaw background tasks into the Agent Status view, so cron runs, background agent spawns, and CLI operations are no longer hidden off to the side. If you rely on overnight jobs or background automations, you can now monitor them from the same place you watch active coding agents.

  • See running, completed, failed, and blocked work in one list.
  • Cancel running background tasks from the sidebar context menu.
  • View task counts in the summary node alongside your agent totals.
  • Keep working normally even if OpenClaw is not installed.

Sessions

Resuming work is now more predictable across the backends Command Central supports today.

Sessions

Resume Session now works across Claude, Codex, and Gemini

If you bounce between AI backends, resume behavior now follows the backend you originally used. Command Central opens the right resume flow, restores the session in the correct project bundle, and gives you quick actions to resume interactively, focus the terminal, or open the transcript.

  • Claude Code sessions resume directly with the correct session match.
  • Codex CLI sessions can jump back in with the latest resume flow.
  • Gemini CLI sessions now get an interactive resume path instead of a dead end.

Terminal And Diff

Reviewing changes and jumping back to a terminal now works better when your launcher setup is imperfect.

Editor

View Diff now opens natively inside VS Code

You no longer need a terminal handoff just to inspect what changed. View Diff now writes Git diff output into a VS Code output channel, which means the command works in more environments and avoids launcher-specific failures.

Terminal

Launcher discovery and fallback behavior are more resilient

When a launcher binary exists in a common install location, Command Central can now discover and reuse it more reliably. When it does not, terminal actions fall back to the integrated terminal instead of failing outright.

  • Successful launcher discovery is cached for faster follow-up actions.
  • Focus Terminal and View Diff now degrade gracefully.
  • A bad primary launcher path no longer blocks fallback resolution.

Notable Fixes

Smaller improvements that tighten up the experience behind the scenes.

Cleaner extension wiring

  • The OpenClaw quick-actions helper is now wired into the extension.
  • That removes a dead export warning and keeps the integration path cleaner.

Infrastructure keeps moving

  • The test suite grew to 1156 passing tests.
  • Dedicated OpenClaw task service files are now part of the codebase.
  • The OpenClaw integration roadmap is documented for the next rollout phases.
