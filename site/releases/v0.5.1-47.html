Command Central Release Notes

This release closes the loop on today’s v0.5.1-46 through v0.5.1-47 work. The headline: discovery is easier to debug, and diff review now feels native inside VS Code instead of detouring through a workaround flow.

Discovery diagnostics Native diff picker OpenClaw tasks + resume still included 1160-test suite
Debug discovery faster

When the sidebar is empty, Command Central now does a better job of telling you why.

Review diffs with native VS Code UX

View Diff now hands off to a proper editor-native diff flow instead of an output-channel bridge.

The current v0.5.1 line is coherent

Auto-discovery, background tasks, smarter resume, launcher fallbacks, and native diff review now fit together as one story.

Discovery Diagnostics

You can now see more clearly why auto-discovery is working, quiet, or partially populated.

Diagnostics

Discovery reports now explain what Command Central checked

If you open Command Central and nothing appears, the diagnostics view now gives you a clearer explanation of the current discovery state. That means less guessing when a supported agent is missing and less time spent wondering whether the issue is your terminal, your launcher path, or the process itself.

  • See whether supported agent processes were detected at all.
  • Tell the difference between "nothing running" and "something was filtered out."
  • Debug launcher or session state without leaving VS Code.

Diff Review

Reviewing what an agent changed now feels more like the rest of the editor.

Editor

View Diff now uses VS Code's native diff picker

You can still jump from the sidebar into an agent's changes, but the final handoff now uses a native VS Code diff flow instead of an output-channel-based fallback. That makes comparison faster to read and closer to what users already expect from source control inside the editor.

  • Keep using the same sidebar quick action.
  • Review changes in a more familiar VS Code-native way.
  • Reduce friction when launcher-specific diff paths are unavailable.

Included In The Current Build

The latest release also carries forward the larger workflow improvements already shipped in this v0.5.1 rollout.

What you already have in v0.5.1

  • Auto-discovery for supported Claude, Codex, and Gemini sessions running outside the launcher.
  • OpenClaw background tasks in the Agent Status sidebar.
  • Backend-aware session resume across Claude, Codex, and Gemini.

Workflow polish that stays with you

  • Launcher fallback behavior is more resilient when your primary path is wrong or missing.
  • The Agent Status tree supports recency-first sorting, project grouping, and relative freshness labels.
  • The current suite size is 1160 tests across 83 files.
